Version 4 changes

Results page

  • Action: Changed microcopy for email alert link.
  • Reason: Feedback from users suggested it would be more useful to receive email alerts/ updates, rather than to receive a copy of the results. After a peer review of the microcopy with other Defra content designers, we decided on ‘get email alerts about new funding’ because: A) it’s shorter, and therefore easier to read than other options, such as ‘sign up for email updates’ B) it explains to users what the email alerts would be for (rather than the specialist publisher microcopy which is ‘get email alerts’).

  • Action: New page.
  • Reason: Functionality to allow users to sign up for email alerts.

Version 3 changes

Start page

  • Action: Added ‘private business’ to bulleted list.
  • Reason: Some funding can also come from private businesses.

Location page

  • Action: Removed list of regions and replaced with a text input with autocomplete. Users can search for a county, town/ city or a catchment.
  • Reason: Asking users to identify a region was too problematic. Some users weren’t sure which region they were in, or what the official region boundaries were.

Map page

  • Action: Removed text at top about showing users national as well as regional results.
  • Reason: Felt this would be confusing and no longer necessary, as we no longer giver users a list of regions on the previous page.

Benefits page

  • Action: Pulled out content with inset text component from the design system.
  • Reason: Users don’t always read the content at the top of the page. We want to highlight to inexperienced users in particular what the purpose of the page is.

Results page

  • Action: Altered content at top of page in second paragraph.
  • Reason: To make it clearer to users that not all funding is specifically for NFM, but may be for other environmental work. Aim is to encourage less experienced users to think carefully about the other benefits of their project when they apply for funding. We want to avoid users thinking they can apply for projects without being able to prove they can deliver in certain areas.
  • Action: Added ‘need more help?’ heading.
  • Reason: Users don’t seem to notice the help page link, so we’ve added a heading to make that content more prominent.
  • Action: Added in ‘get these results in an email’ link.
  • Reason: Users expressed a desire to be able to sign up for email notifications/ see their results in an email.
  • Action: Edited info under each result. No longer shows region. Shows funding areas. External links edited to make them accessible.
  • Reason: As we no longer ask for a region, it seems confusing to show this information. Giving users info about the funding areas to make it easier for them to decide which results look appropriate for them.

Help page

  • Action: Used bullet points to make text at top of page more scannable.
  • Reason: The text was previously all in one paragraph - this should make it easier for users to scan and read.

Full history of design decisions

Version 2 changes

Location page

  • Action: Changed title from 'region' to 'area'
  • Reason: 'Region' can mean different things to different people. ‘Area’ is a more generic term.
  • Action: Added map feature - so users can find location on a map.
  • Reason: Some users weren’t sure which area to choose as they are on a border, or span 2 different areas.

Results page

  • Action: Moved away from specialist publisher. Displaying results in a simpler way.
  • Reason: Users struggled when interacting with specialist publisher layout - they didn’t understand the purpose of the search box or filters.
  • Action: Each result links to further information on the funder’s website, rather than a separate page on GOV.UK about that fund.
  • Reason: It seemed more easier (from a maintenance perspective) to take users directly to the funder’s website, rather than a separate page on GOV.UK about that result.
